#3. Bean
------------------------------------------------
1. xml    vs   Annotation
>> xml : 운영
>> Annotation : 개발


2. @Component
- @Componenet 일반적인 컴포넌트 <bean> 스프링이 관리하는 객체
- @Componenet   구제화된 형식
    @Contoller  웹요청받아서 응답
    @Service    서비스 레이어, 비즈니스 로직
    @Repository 데이터베이스

3. Bean 의존관계주입
    1. @Autowired - 정밀한 의존관계
        - 프로퍼티, setter, 생성자,, 적용
    2. @Qualifier - 동일한 타입의 bean 구분
    3. @Value     - 단순값
    4. @Resource  - 자원연결( .properties )

4. component-scan
<context:component-scan base-package="경로설정"/>
