문제점
db에 전에 입력해뒀던 데이터 값이 남아 있어서 update ~ delete 까지 데이터가 반영이 안된 상황. 

해결방안 
db에서 delete from table이름; 을 이용해서 데이터를 지워주고, 만일 시퀀스 테이블도 생성했다면 drop sequence 테이블명_seq;를 이용해서 시퀀스 테이블도 지워준다. 커밋까지 해주고 나서 다시 실행시켜주면 된다.


느낀점. 
spring / spring boot 를 활용해서 db를 테스트할때, db상에 남아있던 기준의 데이터를 지워주고 나서 다시 실행해야한다는 번거로움이 있지만, db테스트를 안전하게 하기 위해서는 거쳐야하는 과정이라고 생각한다. 계속 배우면서 기존에 남아있던 db 데이터를 안지우고 해도 오류가  나지않는 방법이 있다면, 그 방법으로도 다시 시험해보고싶다.

문제점
Thymeleaf에서 th:href="" 구문을 통해 숫자로 데이터를 받아서 브라우저를 띄우고 싶은데, 
th: 가 빠지게 됨으로써 문자열로 데이터를 받아 오류가 발생한 상황이다.  

해결방안
Thymeleaf에서는 특정 데이터를 받고 싶을때는, th:를 사용해서 원하는 데이터를 받아와야
정상적으로 값이 출력이된다. 

느낀점
기존에는 th:를 사용하지 않고 해서 그런지 많이 익숙하지않지만, 자주 사용하다보면 금방 익숙해질수 있을거라고 생각한다.

---
문제점
dao에서  selectTotalCnt에 해당하는 mapper에서  찾지 못해서 생긴 오류이다.

해결방안
dao와 mapper를 비교해보고 빠져있거나, 추가해야될 부분을 찾아서 수정해주면 된다.

느낀점
service와 dao에 있는 selectTotalCnt가 서로 대소문자가 맞지않아서 dao에서 수정을 해줬다.
dao에서 수정을 했으면, mapper에서도 수정을 해줘야했는데, 수정을 못해서 오류가 발생했기때문에,
dao와 mapper는 서로 관련이 있기 때문에, 어느 하나가 맞지 않으면 실행이 제대로 안되므로
좀 더 주의를 가져야 될 필요성을 느꼈다.

문제
Service에서 db 데이터를 받아올때 매칭이 잘못되어 생긴 오류이다.

해결방안
브라우저를 띄우기 위한 db 데이터를 Service(오류가 발생한 서비스: Oauth2IUserService)를 통해서
