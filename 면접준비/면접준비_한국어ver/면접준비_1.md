    > 객체지향프로그래밍이란?  (키워드: #(데이터+메서드)객체#프로그램 설계&구현 하는 프로그래밍)
    - 데이터(속성, 변수) --(처리 하는)-- 메서드(함수) 
        ㄴ--> 하나로 묶음(독립적인 단위) = ★객체  
    ㄴ --> (★객체) 중심  → 프로그램 설계&구현하는 프로그래밍

    > 현실세계의 개념을 모델링해서 재사용성과 유지보수를 높이는 프로그래밍
    ※ 설치지향 비교했을때 장점 : 코드 재사용성과 모듈화가 뛰어나 유지보수 용이 
    ex.) JAVA

- 객체지향의 4대 특성 
    >캡슐화(Encapsulation) - 데이터, 함수 → 하나의 객체로 묶음 & 외부 직접 접근X → 보호 
    >상속(Inheritance) - (기존 클래스) 속성, 메서드 --(물려받음)--> New 클래스 → 재사용
    > 다형성(Polymorphism) - (같은 이름) 메서드 → (상황에 따라) 다른 방식 동작
    > 추상화(Abstraction) - (불필요한) 세부 구현 숨김 & Only 필요한 기능 --(제공)--> 인터페이스

    > 실제 프로젝트에서 캡슐화: 클래스 변수를 private로 두고 getter/setter 접근

- JVM의 역할

        // 피보나치 수를 구하는 메서드
        public static int fibonacci(int n) {
            if (n <= 1) return n;
            return fibonacci(n - 1) + fibonacci(n - 2);
        }

        public static void main(String[] args) {
            // 2단부터 9단까지
            for (int i = 2; i <= 9; i++) {
                System.out.println("=== " + i + "단 (피보나치) ===");
                for (int j = 1; j <= 9; j++) {
                    // 곱셈 결과 대신 피보나치 수열 값 출력
                    int fib = fibonacci(j);
                    System.out.println(i + " x " + j + " → Fibonacci(" + j + ") = " + fib);
                }
                System.out.println();
            }
        }
    }
